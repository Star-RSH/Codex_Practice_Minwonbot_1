# 삼성생명 민원 지식봇 화면 기획서

작성일: 2026-06-15
대상 화면: 단일 메인 화면
기획 기준: 모바일 우선, 공개·가상·더미 FAQ 기반, 참고용 답변, 확정 판단 금지

## 1. 화면 목표

- 사용자가 민원성 질문을 빠르게 입력하고 답변 초안을 확인한다.
- 답변과 함께 근거 FAQ를 바로 보여준다.
- 금액, 보장 여부, 확정 판단, 범위 밖 질문은 담당자 확인으로 자연스럽게 연결한다.
- 보험/카드 도메인을 자동 감지하되, 사용자가 직접 탭으로 검색 범위를 바꿀 수 있게 한다.

## 2. 정보구조(IA)

모바일 기준 상단에서 하단 순서:

1. 서비스 헤더
- 서비스명: `삼성생명 민원 지식봇`
- 설명 한 줄: `공개 FAQ를 바탕으로 참고용 답변을 안내합니다.`

2. 면책 배너
- 항상 펼침
- 이유: 이 서비스의 책임 범위와 한계를 질문 전 먼저 이해시켜야 함
- 포함 내용: 개인정보 입력 금지, 금액·보장 여부는 담당자 확인 필요, 최종 판단 아님

3. 검색 범위 탭
- 항상 펼침
- 탭: `보험`, `카드`
- 기본값: 자동 감지 결과와 일치하는 탭 활성화
- 사용자가 직접 탭 변경 가능
- 자동 감지 중이어도 사용자가 바꾸면 수동 지정 우선

4. 질문 입력 영역
- 질문 입력 textarea
- 실행 버튼
- 보조 안내 문구
- 개인정보 유사 숫자 차단 안내

5. 상태 영역
- 초기/입력 중/로딩/오류 상태 메시지
- 높이는 고정에 가깝게 유지해 레이아웃 흔들림 최소화

6. 답변 카드
- 결과가 있을 때 최상단 노출
- 답변 본문
- 근거 요약
- `담당자 확인 필요` 배지

7. 근거 FAQ 리스트
- 답변 카드 아래
- 기본 3건 노출
- 모바일에서는 모두 펼침
- FAQ별로 `FAQ #`, 카테고리, 질문, 답변, 출처 표시

8. 담당자 확인 유도 블록
- 결과 하단 고정 위치
- 범위 밖, 금액, 보장 여부, 정보 부족 시 강조

접기/펼치기 원칙:

- 면책 배너: 펼침 유지
- 검색 범위 탭: 펼침 유지
- 질문 입력: 펼침 유지
- 답변 카드: 펼침 유지
- 근거 FAQ 리스트: 모바일에서는 펼침 유지, 추후 PC에서만 접기 검토 가능

## 3. 핵심 흐름

### 기본 흐름

1. 사용자가 질문을 입력한다.
2. 시스템이 개인정보 유사 숫자를 먼저 검사한다.
3. 문제 없으면 로딩 상태로 전환한다.
4. 자동 감지 또는 사용자가 지정한 도메인으로 FAQ를 검색한다.
5. 답변 카드에 참고용 답변과 근거를 보여준다.
6. 금액/보장/확정 판단/범위 밖이면 `담당자 확인 필요` 배지와 확인 안내를 함께 보여준다.

### 세부 흐름

질문 입력 → 유효성 검사 → 로딩 → 답변 생성 또는 검색 결과 표시 → 근거 FAQ 노출 → 담당자 확인 필요 시 안내

### 개인정보 유사 숫자 차단 흐름

1. 사용자가 긴 숫자열 또는 주민번호 형태 숫자를 입력한다.
2. 실행 즉시 차단한다.
3. 서버 호출 없이 입력 영역 아래 오류 문구를 노출한다.
4. 포커스는 입력창에 유지한다.

## 4. 화면 상태 5종

### 4-1. 초기

- 탭 기본 선택 상태
- 질문 입력 전
- 답변 카드 대신 안내 문구 노출
- 근거 FAQ 영역은 빈 상태 카드 표시

노출 문구 예:
- `궁금한 내용을 입력하면 관련 FAQ와 함께 참고용 답변을 보여드립니다.`

### 4-2. 입력 중

- 사용자가 질문을 작성 중
- 실행 버튼 활성화 조건 검토
- 개인정보 유사 숫자 감지 시 즉시 경고
- 도메인 탭은 계속 보이되 선택 변경 가능

### 4-3. 로딩

- 실행 버튼 비활성화
- 버튼 라벨 변경 또는 스피너 표시
- 답변 카드 영역은 스켈레톤 또는 `답변을 준비하고 있습니다`
- FAQ 리스트도 `관련 FAQ를 찾는 중입니다`

### 4-4. 결과

- 답변 카드 노출
- 근거 라인 노출
- FAQ 리스트 노출
- 필요 시 `담당자 확인 필요` 배지 표시
- 자동 감지 결과가 있다면 탭 활성 상태와 일치

### 4-5. 오류·범위 밖

- 입력 오류
- 개인정보 유사 숫자 감지
- 검색 결과 없음
- 시스템 오류
- 범위 밖 질문

오류는 한 블록으로 단정하게 보여주고, 답변 카드와 시각적으로 구분한다.

## 5. 컴포넌트 정의

### 5-1. 질문 입력

- 형태: 멀티라인 textarea
- 목적: 자연어 질문 입력
- 규칙:
- 최소 1자 이상 입력 시 버튼 활성
- 주민번호/계약번호처럼 보이는 긴 숫자 입력 차단
- 엔터는 줄바꿈, 실행은 버튼 우선

### 5-2. 실행 버튼

- 위치: 입력창 하단 우측 또는 모바일 전체폭
- 역할: 질문 전송
- 상태: 기본 / 비활성 / 로딩

### 5-3. 답변 카드

- 구성:
- 제목: `참고 답변`
- 답변 본문
- 근거 라인
- 자동 감지 도메인 요약
- 담당자 확인 필요 배지

### 5-4. 근거 FAQ 리스트

- 항목 구성:
- `FAQ #번호`
- 카테고리
- 질문
- 답변
- 출처

### 5-5. 담당자 확인 필요 배지

- 조건:
- 금액 질문
- 보장 여부
- 확정 판단 필요
- 범위 밖
- 정보 부족

- 스타일:
- 고대비 색상
- 답변 카드 상단 또는 근거 라인 근처 배치

### 5-6. 면책 배너

- 위치: 상단
- 내용:
- 공개 FAQ 기반 참고용 안내
- 개인정보 입력 금지
- 최종 확인은 담당자 또는 약관 필요

## 6. UX 라이팅

톤 원칙:

- 차분하고 신뢰감 있는 금융 서비스 톤
- 단정 대신 안내형 문장 사용
- 사용자를 탓하지 않는 오류 문구

### 버튼

- 기본: `답변 확인`
- 로딩: `확인 중`
- 재시도: `다시 확인`

### 플레이스홀더

- `예: 보험금은 어떻게 청구하나요?`
- `예: 실손 청구에 필요한 서류가 무엇인가요?`

### 빈 상태

- 답변 영역:
- `아직 확인된 답변이 없습니다. 질문을 입력하면 관련 FAQ와 함께 참고용 답변을 보여드립니다.`

- FAQ 영역:
- `관련 FAQ가 여기에 표시됩니다.`

### 입력 오류

- 빈 입력:
- `질문을 먼저 입력해 주세요.`

- 개인정보 유사 숫자:
- `주민번호·계약번호처럼 보이는 숫자는 입력할 수 없습니다. 개인정보는 제외하고 질문해 주세요.`

### 시스템 오류

- `지금은 답변을 불러오지 못했습니다. 잠시 후 다시 확인해 주세요.`

### 결과 없음

- `일치하는 FAQ를 찾지 못했습니다. 정확한 안내를 위해 담당자 확인이 필요합니다.`

### 담당자 확인 안내

- `금액, 보장 여부, 최종 판단이 필요한 내용은 담당자 확인 후 안내해 주세요.`

### 면책 배너 문구

- `이 화면은 공개 FAQ를 바탕으로 참고용 답변을 안내합니다. 개인정보 입력은 제한되며, 금액·보장 여부·최종 판단은 담당자 확인이 필요합니다.`

## 7. 접근성

### 색 대비

- 본문 텍스트와 배경 대비 4.5:1 이상
- 배지와 버튼 텍스트 대비 4.5:1 이상
- 오류 색상은 색만으로 의미 전달하지 않고 아이콘 또는 문구 함께 사용

### 키보드 접근

- 탭 순서:
- 헤더
- 면책 배너 링크 또는 버튼(있는 경우)
- 보험/카드 탭
- 질문 입력
- 실행 버튼
- 답변 카드
- FAQ 리스트

- 포커스 스타일은 시각적으로 명확해야 함
- 로딩 중에도 포커스 유실 금지

### 읽기 순서

- 스크린리더 기준으로 시각 순서와 동일
- 상태 메시지는 `aria-live` 영역 고려
- 오류 문구는 입력 요소와 명확히 연결

### 모바일 우선

- 한 손 사용 고려
- 주요 CTA는 하단 가까운 위치
- 입력창, 버튼, 탭 모두 터치 영역 44px 이상
- 스크롤이 길어져도 답변 카드가 먼저 보이도록 입력 다음 바로 배치

## 8. 모바일 우선 레이아웃 원칙

- 단일 컬럼
- 상단에서 입력, 중단에서 결과, 하단에서 근거
- 탭은 2분할 세그먼트 형태
- 버튼은 전체폭 우선
- FAQ 리스트는 카드 반복형

## 10. 화면 뼈대 제작 기준

기획 합의 후 `index.html` 반영 범위:

- 검색/답변 로직 없이 정적 화면 뼈대 구현
- 상태 5종을 class 전환만으로 표현 가능한 구조
- 개인정보 유사 숫자 입력 차단 스크립트 포함
- 보험/카드 탭 UI 포함
- 로컬 미리보기 기준 모바일 우선 레이아웃 구현

구현 제외:

- 실제 Supabase 검색 연결
- 실제 BizRouter 답변 생성
- 실제 자동 감지 로직

## 11. 합의 확인 포인트

- 답변 카드 우선, FAQ 리스트 후순위 구조
- 면책 배너 상시 노출
- 모바일 단일 컬럼 우선
